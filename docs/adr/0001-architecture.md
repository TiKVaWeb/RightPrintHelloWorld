# ADR-0001: Базовая архитектура проекта

- Статус: Accepted
- Дата: 2026-02-13

## Контекст

Проект стартовал как минимальный скрипт, но цель репозитория - демонстрировать строгую инженерную дисциплину даже на простом сценарии.
Нужно было заложить структуру, которую можно расширять без переписывания всего приложения.

## Решение

Принята слоистая структура:

- `dto.py` - межслойные контракты данных;
- `service.py` - доменная логика и валидация;
- `cli.py` - CLI-адаптер и парсинг аргументов;
- `__main__.py` - точка запуска `python -m`.

Ключевые правила:

- доменная логика не зависит от CLI;
- нет циклических импортов;
- типизация `mypy --strict` для `src` и `tests`;
- поведение фиксируется тестами и coverage gate.

## Последствия

Плюсы:

- проще безопасно расширять функциональность;
- интерфейсы между слоями явно определены;
- легче поддерживать качество и предсказуемость изменений.

Минусы:

- для простого кейса больше файлов и ceremony;
- выше порог входа для новых участников.
