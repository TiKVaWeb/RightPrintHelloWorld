# Contributing

Спасибо за вклад в этот overengineered `Hello, World!`.

## Локальная подготовка

Вариант с `uv`:

```bash
uv sync --group dev
```

Fallback без `uv`:

```bash
python -m pip install -e ".[dev]"
```

## Обязательные проверки перед коммитом

Рекомендуемый путь:

```bash
make check-all
```

Проверка pre-commit:

```bash
make pre-commit
```

## Code style и правила

- Соблюдай `ruff` и `mypy --strict`.
- Не добавляй циклические импорты между модулями.
- Доменная логика должна оставаться в `service.py`, CLI-детали - в `cli.py`.
- Новое поведение обязательно покрывай тестами в `tests/`.

## Коммиты

- Делай маленькие атомарные изменения.
- В описании коммита указывай, что изменено и зачем.
- Не смешивай рефакторинг и функциональные изменения в одном коммите.
